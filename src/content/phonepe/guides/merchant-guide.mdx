# Merchant Guide

This comprehensive guide covers all aspects of managing PhonePe payments as a merchant.

## Getting Started as a Merchant

### Account Setup

To use PhonePe Payment Gateway with your WHMCS installation:

1. Ensure PhonePe module is installed and activated
2. Configure your Merchant ID and API Key
3. Set your preferred payment methods
4. Test with development environment first

### Your Dashboard

<Text variant="body-default-m" onBackground="neutral-weak" marginBottom="12">
Access your payment dashboard through WHMCS Admin Panel:
</Text>

- Navigate to Orders → Payments
- View all PhonePe transactions
- Filter and search transactions
- Access transaction details and history

## Transaction Management

### View All Transactions

<Text variant="body-default-m" onBackground="neutral-weak" marginBottom="12">
Access comprehensive transaction history:
</Text>

1. Go to Orders → Payments in WHMCS
2. Filter by PhonePe gateway
3. View transaction list with details

### Search and Filter

<Text variant="body-default-m" onBackground="neutral-weak" marginBottom="12">
Find specific transactions easily:
</Text>

**Search Options:**
- By Transaction ID
- By Customer Name
- By Invoice Number
- By Amount
- By Date Range

**Filter Options:**
- Payment Status (Success/Failed/Pending)
- Payment Method
- Date Range
- Amount Range
- Specific customer

## Refund Management

### Processing Refunds

<Text variant="body-default-m" onBackground="neutral-weak" marginBottom="12">
How to refund customer payments:
</Text>

<Column gap="10" marginBottom="24">
  <Text variant="body-default-m" onBackground="neutral-weak">1. Go to Orders → Payments</Text>
  <Text variant="body-default-m" onBackground="neutral-weak">2. Find the transaction to refund</Text>
  <Text variant="body-default-m" onBackground="neutral-weak">3. Click "Refund" button</Text>
  <Text variant="body-default-m" onBackground="neutral-weak">4. Enter refund amount (full or partial)</Text>
  <Text variant="body-default-m" onBackground="neutral-weak">5. Add refund reason/note</Text>
  <Text variant="body-default-m" onBackground="neutral-weak">6. Click "Process Refund"</Text>
  <Text variant="body-default-m" onBackground="neutral-weak">7. Confirmation is sent to customer</Text>
</Column>

## Best Practices

### Session Management

<Text variant="body-default-m" onBackground="neutral-weak" marginBottom="12">
Ensure uninterrupted payment processing:
</Text>

- No logout issues during payment processing
- Built-in session management prevents interruptions
- Customers remain logged in through checkout
- Secure session handling protects user data

### Payment Security

<Text variant="body-default-m" onBackground="neutral-weak" marginBottom="12">
Maintain secure payment operations:
</Text>

- Keep PhonePe credentials confidential
- Use strong passwords
- Enable two-factor authentication
- Regularly update WHMCS
- Monitor for suspicious activity

### Customer Experience

<Text variant="body-default-m" onBackground="neutral-weak" marginBottom="12">
Optimize for better results:
</Text>

- Keep checkout process simple
- Display PhonePe clearly as option
- Provide payment confirmation quickly
- Respond to payment issues promptly
- Maintain transparent fee information
