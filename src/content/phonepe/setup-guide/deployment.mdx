## Development to Production Migration

### Step 1: Environment Switch

<Text variant="body-default-m" onBackground="neutral-weak" marginBottom="12">
Switch from development to production:
</Text>

<Column gap="10" marginBottom="24">
  <Text variant="body-default-m" onBackground="neutral-weak">1. Log in to WHMCS Admin Panel</Text>
  <Text variant="body-default-m" onBackground="neutral-weak">2. Go to Settings → Payment Gateways</Text>
  <Text variant="body-default-m" onBackground="neutral-weak">3. Click PhonePe Settings</Text>
  <Text variant="body-default-m" onBackground="neutral-weak">4. Change Environment: Development → Production</Text>
  <Text variant="body-default-m" onBackground="neutral-weak">5. Verify all credentials are correct</Text>
  <Text variant="body-default-m" onBackground="neutral-weak">6. Click Save Changes</Text>
</Column>

### Step 2: Verify Production Credentials

<Card padding="24" radius="l" border="accent-alpha-medium" background="accent-alpha-weak" marginBottom="24">
  <Column gap="16">
    <Text variant="heading-strong-m" onBackground="neutral-strong">Credential Verification</Text>
    <Column gap="8">
      <Text variant="body-default-s" onBackground="neutral-weak">1. Log in to PhonePe Business Account</Text>
      <Text variant="body-default-s" onBackground="neutral-weak">2. Switch to Production mode</Text>
      <Text variant="body-default-s" onBackground="neutral-weak">3. Copy Production Merchant ID</Text>
      <Text variant="body-default-s" onBackground="neutral-weak">4. Copy Production API Key</Text>
      <Text variant="body-default-s" onBackground="neutral-weak">5. Verify both match WHMCS configuration</Text>
      <Text variant="body-default-s" onBackground="neutral-weak">6. Test with small transaction</Text>
    </Column>
  </Column>
</Card>

### Step 3: Verify Callback Configuration

<Text variant="body-default-m" onBackground="neutral-weak" marginBottom="12">
Ensure webhooks are production-ready:
</Text>

- Go to PhonePe Business Account
- Navigate to Webhook Settings
- Verify Callback URL points to production domain
- Test webhook delivery from PhonePe
- Confirm callbacks are being received
- Check WHMCS logs for webhook entries

## Production Deployment

### Step 1: Final Backup

<Text variant="body-default-m" onBackground="neutral-weak" marginBottom="12">
Create complete production backup before deployment:
</Text>

```bash
# Backup database
mysqldump -u username -p whmcs_db > whmcs_backup_$(date +%Y%m%d_%H%M%S).sql

# Backup files
tar -czf whmcs_files_backup_$(date +%Y%m%d_%H%M%S).tar.gz /path/to/whmcs
```

## Security Deployment

### SSL/TLS Configuration

<Text variant="body-default-m" onBackground="neutral-weak" marginBottom="12">
Ensure secure communications:
</Text>

- Install valid SSL certificate
- Configure HTTPS everywhere
- Set HTTP to HTTPS redirect
- Configure security headers
- Test SSL configuration
- Verify certificate validity

### API Security

<Text variant="body-default-m" onBackground="neutral-weak" marginBottom="12">
Protect your PhonePe credentials:
</Text>

- Store credentials securely
- Never hardcode sensitive data
- Use environment variables
- Restrict file permissions
- Regular credential rotation
- Monitor API usage

## Troubleshooting Production Issues

### Payment Processing Issues

<Text variant="body-default-m" onBackground="neutral-weak" marginBottom="12">
If payments fail in production:
</Text>

**Diagnosis**
1. Check transaction logs
2. Verify PhonePe status page
3. Review error messages
4. Check webhook callbacks
5. Verify network connectivity
6. Review server logs

**Resolution**
1. Verify credentials are correct
2. Check callback URL accessibility
3. Review firewall rules
4. Test with different payment method
5. Contact PhonePe support
6. Document issue for review
